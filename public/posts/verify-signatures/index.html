Update on the GPG Signing of My Website

You know that feeling when something just doesn’t sit right? Yeah, having the GPG signature embedded inside my HTML files always bothered me a little. Sure, it was super convenient to validate everything with a simple command like curl https://adlermedrado.com.br/missives/what-id-tell-my-23-year-old-self/ | gpg --verify, but the idea of having the signature “glued” to the original file just felt wrong. It was like the file wasn’t really the original anymore, you know?

So I decided to change that — and here’s why keeping the signatures separate is such a better idea:

1 — Full integrity: The original file stays untouched, with zero “edits” just to include a signature. It remains in its purest form.

2 — Total flexibility: You can sign any file type — HTML, CSS, JSON, whatever you want. And the best part: you can automate the whole process in your pipelines, making everything easier.

3 — Clean and simple verification: No visual noise, nothing breaking parsers, and verification becomes universal and painless. Just run the command and you’re done.

So starting now, the files on my site will no longer carry embedded signatures. And if you want to learn how to verify the authenticity of my HTML, CSS, JSON, and other files, stay tuned — I’ll walk you through everything step by step.

Verify the .asc signatures

Every published asset (HTML, CSS, XML, JSON, etc.) ships with a matching .asc file that holds its detached, ASCII-armored signature. Keep the original file and its .asc together and use GPG to validate integrity and authorship.

  1. Import my public key (only needed once):
    curl -O https://adlermedrado.com.br/pub-key.asc
gpg --import pub-key.asc
  2. Download the file you care about and its signature (example: homepage):
    curl -O https://adlermedrado.com.br/index.html
curl -O https://adlermedrado.com.br/index.html.asc
  3. Verify:
    gpg --verify index.html.asc index.html
    A “Good signature from Adler Medrado” message confirms the file is intact and signed by me. If you see a BAD signature or a different key ID, treat the file as untrusted.

Tip: you can verify any other asset the same way, just pair file.ext with file.ext.asc.
